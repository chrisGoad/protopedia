PrototypeJungle
Things hidden in code

Prototype chains constitute a general structure that underlies all JavaScript computations. So far, they have been hidden in code. But chains might make sense to many non-programmers, if they could see them. After all, the idea that some things are specialized variants of others (firs are connifers are trees) is part of general cognition, not programming expertise. And once visible, they can be manipulated too, giving new powers to non-programmers.

A major purpose of PrototypeJungle is to run this experiment. More is required than the intention to expose chains, if they are embedded in an otherwise tangled structure. It turns out, however, that prototype trees, that is, tree data structures with added information which connects nodes along prototype chains, are adequate in functionality, and sufficiently constrained to be presented effectively in a UI. They can be saved as components too.

There have been other cases of exposing the structure underlying related sets of computations via a UI. The cannonical example: consider a piece of code that performs a financial computation compared to a spreadsheet with the same purpose. Exactly the same computational steps might be involved in a calculation, but by exposing the structure outwardly, powers are given to everyone.

Another example, recently in vogue, though with a long history: Flow-based programming arises from the observation that many kinds of computation can be formulated as flows of messages between active nodes of a graph, and from the utility of making the graph visible.

These cases illustrate that the effort of bringing structure out of the opaquenness of code can bring large rewards.
