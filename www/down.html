PrototypeJungle
Tools for building inspectable JavaScript applications

Prototype Jungle is down for a very brief maintainence, but the static documents are available.

Development status: proof of concept. Open source, under an MIT license

The core idea: PrototypeJungle applications are built by successive specialization of tree structures. Central to this process is a tree-instantiation operation which is sort of like tree copy, except that the new nodes inherit prototypically from their counterparts in the old tree as appropriate. The steps of specialization are themselves components in the system. The components can be stored separately, and assembled in code or via a UI.

The application area: initial experiments involve data display using HTML5 canvas. In this domain, trees represent computationally defined graphical objects, ranging in complexity from simple shapes, up through complete infographic assemblies (eg charts, flow diagrams). Examples appear below, as shown in PrototypeJungle's inspector.

This inspector follows the lead of the DOM inspectors that come with all modern browsers in that clicking on things displays the information structures which underly them - in this case JavaScript objects with their prototype chains and functions. The objects can be edited and saved in their modified form. Among other things, this allows applications to be customized by non-programmers.
