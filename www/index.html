JavaScript Prototypes in the Open

Prototypes play a central role in JavaScript, but are unseen except by coders. Prototype chains are neither stored in JSON, nor exposed systematically in user interfaces1.

There is a reason: lack of uniform structure of the object graph within which the prototype chains appear. However, PrototypeJungle provides a way of building structures with the required regularity via its component system. The scheme results in highly adjustable applications. The chart, when viewed in the UI, manifests complete adjustability in that each instance and prototype is subject to modification.

The component system relies on hierarchic prototypes. In normal JavaScript usage, the instantiation of a prototype (creation of a new object which inherits from the prototype) is a one-level operation, implemented by Object.create or new. PrototypeJungle extends instantiation to hierarchic structures, so that trees of arbitrary depth serve as templates from which instances can be spawned at one blow.

The structures in question may incorporate functions and internal prototype chains. Serialization is also supported. Together, these capabilities yield a component system in which applications are built by instantiation and assembly from stored elements. After instantiation, the components, being prototypes, are still "live" - that is, any adjustments are inherited by instances.

The assemblies have the uniformity which allows the prototypes chains to be exposed in a user interface.
