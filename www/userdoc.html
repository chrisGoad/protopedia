1. Customizing items

This document explains how to take PrototypeJungle items, such as charts, customize them to your own needs, and bind data to them. To save items that you have customized in the inspector you need to sign in. No registration is needed; you can sign in with your email or twitter account.

First, load the chart of interest into the inspector. PrototypeJungle is new, so at the moment the catalog of possibilities is very small;only the following three varieties are avaliable:

Then adjust prototypes and instances in the chart as explained at the bottom of X, and insert captions as desired. You can bind your own data to the chart at this stage, or take care of that aspect in a later step. To bind data at this stage, toggle open the data source, and enter the Url from which you wish to fetch the data. When your variant is ready, save it; it will be stashed in the area allocated to you when your first signed in.

If you have already bound data to the item, and have no need for later changes (eg programmatic updating of data, perhaps reflecting a real-time source), then the chart is all ready for embedding in web pages (click on the share button in the inspectofr for the exact embedding code).

1. Loading items with JavaScript

For more flexibility, items can be loaded into a web page with code such as this: 

<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
</head>
<script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/jquery/2.0.3/jquery.min.js">
</script> 
<script type="text/javascript" src="http://s3.prototypejungle.org/min/draw.js"></script>
<script>

// load a bar chart,and bind data
$('document').ready(function () {
   var draw = prototypeJungle.draw;
   var itemUri = "http://s3.prototypejungle.org/sys/repo0/chart/variants/BarChart/v4";
   var cnv = draw.initCanvas($('#canvas'));
   cnv.bkColor = "white";
   cnv.fitFactor = 0.7;
   draw.installAsRoot(itemUri,cnv,function (rs) {
     rs.setData({value:[{x:1,y:26},{x:2,y:12},{x:3,y:40},{x:4,y:14}]});
     rs.update();
     cnv.fitContents();
     cnv.refresh();
   });
});
</script>
<div> Example: barchart </div>
<div id="canvas" style="width:800px;height:500px;border:solid thin black"></div>
</body>
</html>

The itemUri should be the URI of the variant that you saved off, which you can discover from the share button.

1. Loading data

As seen from the above example, setData does the binding of data to an item. The following code fragment loads and binds data programmatically: 

prototypeJungle.om.loadData("http://s3.prototypejungle.org/sys/repo0/data/bardata0.json",function  (e,d) {
  if (e) {
      ... handle error eg alert(e.message); ...
  } else {
      item.setData(d);
  }
}

For each kind of item you can see by example what sort of data it expects from the data button in the top bar. Of course, using this primitive, you could programmatically load and display data at intervals from a changing data source.
